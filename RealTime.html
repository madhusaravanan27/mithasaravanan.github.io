Real Time Sports Analytic Platform

Project Overview

The Real Time Sports Analytic Platform is a distributed data pipeline designed to process a live lacrosse game stream, update player and team statistics, and generate live box scores for web developers. This system integrates various technologies, including PySpark, MongoDB, Microsoft SQL Server (MSSQL), and Minio (S3-compatible storage), to simulate a real-world sports analytics environment.

Technologies Used

  • PySpark: For distributed data processing and transformations.
  • MongoDB: NoSQL database for storing real-time box scores.
  • Microsoft SQL Server (MSSQL): For storing team and player reference data.
  • Minio: S3-compatible object storage for live game stream data.
  • Apache Drill: Distributed SQL query engine for querying integrated datasets.
  • Jupyter Lab: For developing and running PySpark code.

Key Accomplishments

1. Real-Time Data Stream Processing

Developed a pipeline to ingest a live game stream from an S3 bucket (Minio) containing shot events and goals during a lacrosse game. Using PySpark, the pipeline tracks player shots, goals, and team scores as the game progresses.

2. Box Score Generation

Created a live box score updated in real-time, transforming game events into a JSON format suitable for web display. The box score is updated periodically during the game and stored in MongoDB for quick access by web developers, featuring player and team statistics such as shots taken, goals scored, and current game status.

3. Player and Team Statistics Update

After the game concludes, the pipeline updates the team win/loss records and player statistics in new MSSQL tables (players2 and teams2) to reflect final game results. PySpark is used to calculate cumulative shots and goals, ensuring accurate reflection of game outcomes in the database.

4. Database Integration and Querying

Integrated Apache Drill to query across distributed datasets from MSSQL, MongoDB, and Minio. Wrote SQL queries in Apache Drill to retrieve and display data on teams, players, and the live game stream for in-depth analytics and reporting.

5. End-to-End Data Pipeline

Combined all components into a comprehensive PySpark script to automate the entire process: ingesting the game stream, updating statistics, generating live box scores, and finalizing stats post-game. This ensures the pipeline runs consistently during the game, providing web developers with up-to-date statistics as the game progresses.
